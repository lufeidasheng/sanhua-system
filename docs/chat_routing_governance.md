# 三花聚顶系统 · 聊天路由治理说明

## 目的

本文档用于统一三花聚顶系统当前的聊天路由治理口径，避免不同入口、不同模块、不同声明文件对“谁是正式主聊天桥”产生歧义。

本文档只定义治理口径，不替代底层实现说明。

---

## 当前治理结论

### 1. 正式主聊天桥
**`ai.chat`**

定义：
- `ai.chat` 是系统当前的正式主聊天桥。
- 新的聊天入口、统一动作入口、默认聊天路径，应优先接入 `ai.chat`。
- 当系统需要定义“默认第一跳聊天动作”时，应以 `ai.chat` 为准。

---

### 2. 内部/兜底桥
**`AICore.chat / aicore.chat`**

定义：
- `AICore.chat` 是 AICore 的内部对话服务能力。
- `aicore.chat` 是对 `AICore.chat` 的动作包装与兼容入口。
- 二者主要承担以下角色：
  - AICore 内部服务调用
  - 历史兼容入口
  - 正式主聊天桥失败时的兜底路径

约束：
- `AICore.chat / aicore.chat` 不作为默认正式主聊天桥。
- 不应在文档、manifest、注释中继续将 `aicore.chat` 描述为系统默认正式聊天动作。

---

### 3. 非默认探测分支
**`AICore.ask`**

定义：
- `AICore.ask` 仅保留为非默认探测分支、保留分支或特殊能力分支。
- 可以存在，但不应作为默认聊天第一跳。
- 入口若需要使用 `AICore.ask`，应明确说明其为保留/探测用途，而不是正式主链。

---

## 路由口径

在不命中本地记忆短路的前提下，默认聊天顺序应统一为：

1. `ai.chat`
2. `AICore.chat`
3. `aicore.chat`
4. `AICore.ask`（保留但非默认）

说明：
- 本地记忆短路属于聊天前置短路能力，不属于本文件定义的“正式主聊天桥”竞争范围。
- 本文件只约束非本地记忆命中后的聊天桥顺序与角色。

---

## 口径适用范围

本口径适用于：

- GUI 聊天入口
- 聊天编排器
- 动作注册说明
- manifest 描述
- 相关治理文档
- 后续新增聊天入口的默认路由定义

---

## GUI 启动口径（简版）

- 默认推荐 GUI 启动方式：`./run_gui.sh`
- 正式 GUI 主入口：`entry/gui_entry/gui_main.py`

---

## 不在本文件处理范围内的事项

以下事项不属于本文档处理范围：

- `ai.chat` 的底层桥接实现
- `AICore.chat` 的底层 backend 实现
- dispatcher / model_engine / backend 重构
- action 重命名
- 模型服务链合并
- 测试策略调整

这些属于实现层或后续治理事项，应在独立工单中处理。

---

## 维护要求

后续若出现以下情况，必须同步更新本文档：

- 聊天默认第一跳发生变化
- `aicore.chat` 的角色定义发生变化
- `AICore.ask` 从保留分支提升为正式链路
- 新增新的正式聊天桥

---

## 当前统一结论（一句话版）

**正式主聊天桥是 `ai.chat`；`AICore.chat / aicore.chat` 是内部/兼容/兜底桥；`AICore.ask` 保留但不作为默认第一跳。**
