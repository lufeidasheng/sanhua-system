# 三花聚顶 v1 方向定义

## 系统正式定义
- 三花聚顶不是普通 AI 助手，而是一个本地优先、全天候常驻、具备长期记忆、聚感、行动控制、自我演化以及未来多设备/家居接入能力的数字生命操作层。

## v1 目标定义
- v1 目标：贾维斯的本地内核雏形
- 当前优先做到：
  - 知道用户是谁
  - 知道自己当前状态
  - 能稳定控制电脑核心能力
  - 能记住历史
  - 能持续接管动作
  - 能稳定表达自己的变化

## 六层能力树
- 1. 人格与记忆层
- 2. 聚感层
- 3. 决策与执行层
- 4. 电脑控制层
- 5. 家居中控层
- 6. 自我演化层

## 四阶段路线图
- 第一阶段：本地数字中控成型
- 第二阶段：电脑操作代理成型
- 第三阶段：聚感成型
- 第四阶段：家居与实体世界接入

## 当前阶段重点
- 当前阶段：第一阶段，本地数字中控成型
- 当前主施工方向：
  - 收口主链 live owner
  - 固化状态真相源与可观测性
  - 补齐电脑核心能力闭环
  - 让记忆、聚感、动作接管形成稳定最小闭环

## 当前不做 / 后置项
- 高风险家居控制默认后置，不进入 v1 主施工范围
- 实体设备自动联动默认后置，不在当前阶段直接放开
- 大范围 legacy 清理、GUI 大重构、聊天主链重写默认不做
- 未来多设备/家居接入只保留方向，不在当前工单中抢跑实施

## 统一存储层 / NAS 模块组方向
- NAS 更适合作为未来模块组 / 统一存储服务层，而不是单一功能模块
- 未来命名可在 `storage_hub` / `nas_gateway` 之间收敛
- 该层位于电脑控制层与家居接入层之间，承担统一存储、媒体资源、快照备份、多设备数据底座角色
- 当前阶段只做方向预留，不进入主施工，不抢 ai.chat 主桥与状态真相源主线
- 进入 NAS MVP 审计前，需先压实以下底座：
  - ai.chat 主桥
  - system.health_check / system.status 真相源
  - 用户身份与记忆前置
  - 统一资源接口 / 存储抽象
