# AGENTS.md

## 项目阶段
- 三花聚顶本地稳定版 v1

## 最高优先级原则
- 优先保证主链稳定，不做大范围重构
- 优先修真相源、调用链、状态协议、最小短路
- 默认最小工单驱动
- 一次只解决一个明确问题

## Codex 执行边界
- 默认只改单个文件，除非工单明确允许多文件
- 先审计，再下最小修复
- 不擅自扩展范围
- 不擅自改 GUI 大结构
- 不擅自改聊天主链大结构
- 不擅自进入 legacy 大清理
- 不擅自做自动化大重构

## 关键真相源
- system.health_check 为运行面真相源
- 模块状态/原因/处置相关问答优先基于 system.health_check
- chat_orchestrator.py 已有模块状态/原因/处置短路与规则表

## 输出要求
- 每张工单完成后默认输出：
  - 修改了哪些行
  - 最小 diff 摘要
  - 验证命令
  - 验证结果

## 禁止事项
- 不运行 /review，除非明确要求
- 不因为顺手修改无关模块
- 不在一个工单里混入多个目标
- 不把建议写成长篇空话

## 风格要求
- 结论优先
- 高信息密度
- 最小改动
- 验收可执行
